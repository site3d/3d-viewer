## Оценка Potree для СОД и возможная архитектура

Potree подходит как визуализационный контур СОД для тяжелых 3D-данных (LiDAR / лазерное сканирование) и быстрых сравнительных сценариев, но обычно требует внешнего доменного слоя для управления данными, версиями и правами.

### Что Potree делает хорошо

- Быстрый рендер крупных облаков точек.
- Нативная работа с форматами Potree (`cloud.js`) и EPT (`ept.json`).
- Базовые инструменты измерений, навигации, стилизации и классификаций.
- Простая интеграция в web-приложение как viewer-слой.

### Ограничения Potree как ядра СОД

- Нет полноценной модели СОД: роли, согласования, аудит.
- Нет встроенного каталога активов, версионирования и связей между документами/моделями.
- Потоковая доставка больших данных и кэширование ложатся на внешнюю платформу.
- Для enterprise-сценариев нужны отдельные сервисы безопасности, API-шлюз и метаданные.

### Рекомендуемая архитектура приложения на базе Potree

1. 3D-просмотрщик (frontend):
   - React + Next.js + Potree.
   - Синхронизация камер, режимы рендера, сравнение ревизий.

2. API-слой:
   - Маршрутизация к наборам данных, контроль доступа.
   - Агрегация метаданных проекта, версий, статусов проверки.

3. Дата-слой:
   - S3-хранилище данных для облаков точек.
   - Каталог метаданных (PostgreSQL).
   - Очереди и пайплайн предобработки (конвертация в Potree/EPT, валидация).

4. Доменный слой (бизнес-правила):
   - Управление ролями и проектными пространствами.
   - Ревизии на геометрию.
   - Интеграция с BIM, документами и отчетностью.

Итого: Potree лучше использовать как мощный 3D-просмотрщик в составе СОД.
